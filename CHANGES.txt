OmniPeak Changelog
===============

Here you can see the full list of changes between each release.

Version 1.5
--------------------

Released on Jun 18th, 2026

- Improved detailed analysis reporting `--deep-analysis` - plot all the algorithm important characteristics
- Reworked gap estimation as threshold-free log-space convexity of the candidates-by-gap curve - small gap for narrow marks, larger for fragmented broad marks
- Bugfixes

Version 1.4
--------------------

Released on Jun 3rd, 2026

- Reworked gap estimation as a convergence point of the candidates' number while increasing gap
- Filter out summits by significance
- Clean up .npz coverage files when no `--keep-cache` is set
- Fixed `--bigwig` output after .npz coverage cache cleanup
- Reworked peaks significance estimation when PEPs only are available


Version 1.3
--------------------

Released on November 19th, 2025

- Added `--no-control-regression` command line option to disable control regression from the signal data
- Bug fixes

Version 1.2
--------------------

Released on November 5th, 2025

- Improved summits computation - use Hartigan Dip test to locate single mode peaks and average width
  followed by Gaussian smoothing for summits detection
- JBR integration shows the summits of the peak calls
- Refactorings to improve peak calling code readability and maintainability
- Bug fixes

Version 1.1
--------------------

Released on October 16th, 2025

- Improved peak calling from bigWig files
- Simplified the fragmentation compensation gap computation
- Bug fixes

Version 1.0
--------------------

Released on July XX, 2025

- Initial release
